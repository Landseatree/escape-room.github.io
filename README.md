# 謎解きウェブサイト 構成説明

## ファイル構成

```
escape-room/
├── index.html      … 画面構造（待機／問題／正解／脱落の4画面すべてを含む）
├── style.css        … 全画面のデザイン（このファイル1つにまとまっています）
├── script.js         … 画面遷移・タイマー・キーパッド入力・正誤判定のロジック
└── no1/
    ├── question1.jpg … 謎解き問題の画像（仕様通りのファイル名）
    └── answer1.txt    … 正解の文字列（仕様通りのファイル名、ひらがなで1行）
```

コードは「分けてもよい」とのことでしたが、今回は依存関係を単純にし、
タブレットでの運用時にファイルの迷子が起きないよう、
**HTML・CSS・JSをそれぞれ1ファイルずつ**に分けてあります。
画面（待機・問題・正解・脱落）はすべて `index.html` の中に `<section>` として
用意し、JSが表示/非表示を切り替える1ページ構成です。

## 使い方

1. `no1/question1.jpg` を実際の問題画像に差し替えてください（現状はサンプルのプレースホルダー画像です）。
2. `no1/answer1.txt` を実際の正解（ひらがなのみ、改行や余分なスペースなし）に書き換えてください（現状は「こたえ」というサンプルです）。
3. `index.html` をタブレットのブラウザで開きっぱなしにしてください。
   - スマホ/タブレットの**自動ロック・スリープをオフ**にしておくことを推奨します。
   - file:// で直接開くと `fetch` でのテキスト読み込みがブラウザによってはブロックされる場合があるため、
     簡易サーバーを立てて `http://` 経由で開くことを推奨します（例: `python3 -m http.server` をこのフォルダ内で実行）。

## 問題を追加する場合（no2, no3…）

`script.js` の冒頭にある `PUZZLES` 配列に追記してください。

```js
const PUZZLES = [
  { folder: 'no1', image: 'question1.jpg', answerFile: 'answer1.txt' },
  { folder: 'no2', image: 'question2.jpg', answerFile: 'answer2.txt' }, // 追加例
];
```

現状は仕様書通り1問構成（`currentPuzzleIndex` が常に0）になっています。
複数問を連続して出題したい場合は、正解画面から待機画面に戻る代わりに
次の問題画面へ進むよう `goToCorrect()` 内の処理を調整してください。

## 仕様との対応メモ

- 制限時間: 1人=90秒 / 2人=75秒 / 3人=60秒（待機画面のボタンの `data-time` 属性で管理）
- 不正解時は×マークを2秒表示し、入力をクリア。タイマーは止めずに進行し続けます。
- 残り時間が0になると自動的に脱落画面（青背景）に遷移します。
- 正解すると正解画面（赤背景）が5秒表示された後、自動で待機画面に戻ります。
- 脱落画面も5秒表示後、待機画面に自動で戻る仕様にしています（仕様書に明記がなかったため、
  運用上スタッフが次の人数ボタンをすぐ押せるよう待機画面へ戻す形にしました。
  もしスタッフの手動操作で次に進めたい場合はご相談ください）。
